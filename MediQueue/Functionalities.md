Project Idea: MediQueue
Clinic Appointment & Token Management System
Core Functionalities
1. User Registration (Patient/Doctor/Admin)
○ Patients, doctors, and clinic staff can create accounts with personal
details.
○ Secure password-based authentication.
2. Login System
○ Separate login panels for:
▪ Patients
▪ Doctors
▪ Reception/Admin
3. Profile Management
○ Users can edit:
▪ Name, contact details
▪ Profile picture
▪ Medical history (patients)
▪ Specialization and schedule (doctors)
4. Online Appointment Booking
○ Patients can:
▪ Select doctor
▪ Select date and time slot
▪ Confirm appointment
5. Token Generation System○ After booking, a unique token number is generated.
○ Tokens are displayed on a queue screen.
6. Estimated Waiting Position
○ Display of:
▪ Current token being served
▪ Number of patients ahead in queue
▪ Patient’s token number
7. Missed Appointment Recovery System (Updated Walk-In)
○ If a patient misses their appointment:
▪ The appointment status is marked as “Missed”.
▪ The patient may visit the clinic on the same day.
▪ Reception can add the patient to the waiting list, subject to availability.
▪ The system updates queue position dynamically.
8. Doctor Dashboard
○ Doctors can:
▪ View today’s appointments
▪ Call next patient
▪ Mark appointment as completed
▪ Add patient remarks
9. Admin Dashboard
○ Admin can manage:
▪ Doctors
▪ Patients
▪ Appointments
▪ System settings
▪ Approve/reject leave requests
▪ View system statistics
10. Appointment Rescheduling & Cancellation
○ Patients can:
▪ Reschedule appointment
▪ Cancel appointment
○ Token queue updates automatically.
11. Leave Management System
○ Doctors can:
▪ Apply for leave
▪ Mention reason (e.g., out of city)
○ Admin can:
▪ Approve or reject leave requests
○ If leave is approved:
▪ All affected appointments are automatically cancelled.
▪ Patients receive notification.
▪ The doctor receives a summary of cancelled appointments.
12. Patient History & Visit Statistics
○ System maintains:
▪ Patient name and CNIC
▪ Number of visits
▪ Successful visits
▪ Missed visits
▪ Doctor remarks history
○ System displays:
▪ Total appointments
▪ Visit statistics
▪ Complete appointment history
13. Blacklist Automation System
○ If a patient misses more than 3 appointments:
▪ The system automatically marks the patient as “Blacklisted”.
▪ Blacklisted patients cannot book new appointments.
▪ Admin can manually review or remove blacklist status.
14. Fee & Billing System
○ System manages consultation fees.
○ After a successful appointment:
▪ A fee slip is generated.
▪ Fee slip is downloadable.
▪ Fee slip is printable.
○ System maintains payment records and billing history
